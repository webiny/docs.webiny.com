## Development

### Introducing the `webiny-mcp` Standalone Binary ([#5046](https://github.com/webiny/webiny-js/pull/5046))

MCP configuration was previously handled via the `webiny configure-mcp` command in the Webiny CLI, tying it to Webiny projects. The functionality has been extracted into a dedicated `@webiny/mcp` package with a standalone `webiny-mcp` binary that works in any project, not just Webiny. The package no longer depends on `@webiny/cli-core` or React at runtime.

As part of this change:

- The `webiny configure-mcp` command has been removed — use `webiny-mcp` directly instead
- Added support for Kiro IDE configuration
- Added support for OpenCode IDE configuration ([#5021](https://github.com/webiny/webiny-js/pull/5021))

New focused architect skills replace the monolithic `full-stack-extensions` skill:

- `webiny-full-stack-architect` — for end-to-end extension development
- `webiny-api-architect` — for backend/API extensions
- `webiny-admin-architect` — for admin UI extensions
- `webiny-api-cms-custom-field-type` — covers building custom CMS field types with `DataFieldBuilder`, `FieldType.Factory`, and `FieldBuilderRegistry`

### Improved AWS Credentials Validation ([#5023](https://github.com/webiny/webiny-js/pull/5023))

The CLI now performs more thorough AWS credentials validation during project setup and deployment, providing clearer error messages when credentials are missing or misconfigured.

### Fixed Tailwind CSS Watcher Not Tracking the `/extensions` Folder ([#5041](https://github.com/webiny/webiny-js/pull/5041))

If you were writing Tailwind classes inside the `/extensions` folder, those classes would not be recognized by Tailwind and would be missing from the final CSS output. This has been fixed — the Tailwind watcher now scans the `/extensions` folder alongside Webiny's own packages.

### Fixed Extension Import Deduplication ([#5017](https://github.com/webiny/webiny-js/pull/5017))

Installing an extension that required an import from `webiny/extensions` would create duplicate import statements even if that import already existed. The CLI now correctly appends to existing imports:

```typescript
// Before: duplicate imports
import { Admin } from "webiny/extensions";
import { Api } from "webiny/extensions";

// After: merged import
import { Admin, Api } from "webiny/extensions";
```

Additionally, `extension.json` now supports a `components` array to allow installing multiple components at once.

### Expanded DevTools Panel ([#5034](https://github.com/webiny/webiny-js/pull/5034), [#5032](https://github.com/webiny/webiny-js/pull/5032))

The Webiny DevTools browser extension has been significantly enhanced:

- Added control over sections and views visibility
- Added "copy value" and full object "copy" capabilities for debugging
- Moved Website Builder state inspector to DevTools with real-time updates
- Added license inspector panel
- Added CMS model and entry inspection sections
- Added search/filter functionality across all panels

## Webiny SDK

### New `tenantManager` Methods ([#5018](https://github.com/webiny/webiny-js/pull/5018))

Added SDK methods for tenant management operations:

```typescript
import { sdk } from "webiny/sdk";

// Install a new tenant
await sdk.tenantManager.installTenant({
  tenantId: "69b95c6b1c84b00002f7ff2a"
});

// List all tenants
const tenants = await sdk.tenantManager.listTenants();
```

### New `fileManager` Methods ([#5018](https://github.com/webiny/webiny-js/pull/5018))

Added comprehensive file management SDK methods:

```typescript
import { sdk } from "webiny/sdk";

// Upload a file
const file = await sdk.fileManager.createFile({
  data: fileBuffer,
  name: "document.pdf",
  type: "application/pdf"
});

// List files with filtering
const files = await sdk.fileManager.listFiles({
  where: { type: "image/png" },
  limit: 50
});
```

### CMS Search and Meta Fields Support ([#5009](https://github.com/webiny/webiny-js/pull/5009))

The SDK now supports full-text search queries and meta field retrieval for CMS entries:

```typescript
import { sdk } from "webiny/sdk";

// Search entries with meta fields
const entries = await sdk.cms.entries.list("article", {
  search: "webiny serverless",
  fields: ["title", "content", "meta.createdOn", "meta.savedOn"]
});
```

### New `useBuildParams` Hook ([#5046](https://github.com/webiny/webiny-js/pull/5046))

A new presentation hook exposes build parameters to React components:

```typescript
import { useBuildParams } from "webiny/admin/build-params";

function MyComponent() {
  const { environment, version } = useBuildParams();
  return <span>Running {version} in {environment}</span>;
}
```

### Exported `FieldTypeValidator` Namespace ([#5046](https://github.com/webiny/webiny-js/pull/5046))

All validator interfaces are now exported via the `FieldTypeValidator` namespace, enabling type-safe custom field type authoring:

```typescript
import { FieldTypeValidator } from "webiny/api/cms/model";

const myValidator: FieldTypeValidator.MinLength = {
  name: "minLength",
  validate: (value, settings) => value.length >= settings.value
};
```

## Website Builder

### Configurable Element Overlay Actions ([#5032](https://github.com/webiny/webiny-js/pull/5032))

Extensions can now inject custom actions into element overlays in the editor:

```typescript
import { EditorConfig } from "webiny/website-builder";

export const Extension = () => {
  return (
    <EditorConfig>
      <EditorConfig.ElementOverlay>
        {({ elementId, box }) => (
          <MyCustomButton elementId={elementId} position={box} />
        )}
      </EditorConfig.ElementOverlay>
    </EditorConfig>
  );
};
```

New hooks are available for overlay state:

- `useElementOverlay` — access current overlay state
- `useHighlightedElement` — get the currently highlighted element

### Element Constraints and Lifecycle Hooks ([#5020](https://github.com/webiny/webiny-js/pull/5020))

Component manifests now support a constraints system for validating element placement:

```typescript
const manifest = {
  name: "hero-section",
  constraints: {
    allowedParents: ["root", "container"],
    maxInstances: 1
  },
  descendantConstraints: {
    allowedChildren: ["heading", "paragraph", "button"]
  },
  canDelete: (element, context) => !element.locked,
  onChange: event => {
    // React to element create/update/delete
  },
  onDescendantChange: event => {
    // React to changes in child elements
  }
};
```

New editor hooks for element manipulation:

- `useCreateElement`
- `useDeleteElement`
- `useUpdateElement`
- `useElementInputs`

### Editor Read-Only Mode ([#5010](https://github.com/webiny/webiny-js/pull/5010))

The editor now supports a read-only mode with config components for conditional UI rendering:

```typescript
import { EditorConfig } from "webiny/website-builder";

export const Extension = () => {
  return (
    <EditorConfig>
      <EditorConfig.IsReadOnly>
        <ViewOnlyToolbar />
      </EditorConfig.IsReadOnly>
      <EditorConfig.IsNotReadOnly>
        <FullEditingToolbar />
      </EditorConfig.IsNotReadOnly>
    </EditorConfig>
  );
};
```

Additional improvements:

- New `useEnvConfig` hook for accessing environment configuration
- Improved page editor layout calculations using `data-affects-preview` DOM attribute
- Added `fill-grid` CSS utility class for grid backgrounds

### Configurable Page Settings with View Modes ([#5029](https://github.com/webiny/webiny-js/pull/5029))

Page settings are now fully configurable through a composable API:

```typescript
import { EditorConfig } from "webiny/website-builder";

export const Extension = () => {
  return (
    <EditorConfig>
      <EditorConfig.PageSettings.Group name="seo" label="SEO">
        <EditorConfig.PageSettings.Element name="meta-description">
          <MetaDescriptionField />
        </EditorConfig.PageSettings.Element>
      </EditorConfig.PageSettings.Group>
      <EditorConfig.PageSettings.ViewMode.Drawer />
    </EditorConfig>
  );
};
```

View modes available:

- `PageSettings.ViewMode.Dialog` — opens settings in a modal dialog
- `PageSettings.ViewMode.Drawer` — opens settings in a slide-out drawer

A new `useHotkeys` hook is also available for keyboard shortcut handling.

### Additional Website Builder Utilities ([#5050](https://github.com/webiny/webiny-js/pull/5050))

Added extra utility functions to the Website Builder SDK for common operations during element manipulation and rendering.

## Admin

### Programmatic Route Transition Guards ([#5049](https://github.com/webiny/webiny-js/pull/5049))

The `NavigationPrompt` React component has been replaced with a programmatic API for blocking route transitions:

```typescript
import { useRouter } from "webiny/admin";

function MyForm() {
  const router = useRouter();
  const [isDirty, setIsDirty] = useState(false);

  useEffect(() => {
    const removeGuard = router.addTransitionGuard({
      guard: () => isDirty,
      onBlocked: () => {
        // Show confirmation dialog
      }
    });
    return removeGuard;
  }, [isDirty]);

  const handleSave = async () => {
    await saveData();
    router.unblockTransition(); // Allow navigation after save
    router.navigate("/success");
  };
}
```

This approach is more reliable than the previous component-based solution, which could miss dirty state changes during the React render cycle.

### Fixed Config Re-render Causing Component Remounts ([#5047](https://github.com/webiny/webiny-js/pull/5047))

When a `Config` parent component re-rendered, it could cause inner components to unmount and remount repeatedly, breaking priority-based ordering in the configuration system. The `CompositionStore` now performs atomic replacements of decorators, preventing transient states where both old and new HOCs exist simultaneously.
