## Development

### Fixed Tailwind CSS Watcher Not Tracking the `/extensions` Folder

The Tailwind CSS watcher was only scanning Webiny's own packages for class usage, leaving the `/extensions` folder untracked. Any Tailwind classes used exclusively in custom extensions were stripped from the final CSS output. The watcher now includes the `/extensions` folder, so all classes used in your extensions are correctly detected and included.
