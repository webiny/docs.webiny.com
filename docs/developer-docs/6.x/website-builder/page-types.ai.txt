Understanding:
PageType registration works through usePageTypes() which manages a Map of name -> PageType entries. addPageType adds an entry, removePageType removes by name. The CreatePage dialog renders the selected type's `element` inside a form context. On submit, `formData.properties` and `formData.extensions` and `formData.metadata` are spread into CreatePageParams, with `metadata.pageType: type` always set. This means the form's Bind fields under properties.* become page properties.

The `element` is rendered inside a `<Grid>` that already wraps the type selector, so Grid.Column is used directly (no nested Grid needed).
