Understanding:
PageSettingsGroup.Interface requires: name, label, buildForm(), mapToForm(), mapFromForm(). Optional: description, icon.
PageSettingsGroupModifier.Interface requires: group (target group name), modifyForm(). Optional: mapToForm(), mapFromForm().
The buildForm/modifyForm methods use a form builder API with fields() for defining field types and layout() for positioning.
Data mapping always goes through doc.extensions — doc.properties is reserved for system properties (title, path, snippet, image, tags, seo, social).
Layout positioning in modifiers uses .after("fieldName") to place new fields relative to existing ones.
Both implementations and modifiers are registered via container.register() in a createFeature() block — same DI pattern as other admin extensions.
