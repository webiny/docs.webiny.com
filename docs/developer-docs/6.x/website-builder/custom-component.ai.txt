Understanding Editor Components:
- Two parts: React component + manifest (name, label, group, inputs)
- React component receives inputs prop typed with ComponentProps<YourInputs>
- Manifest tells editor what inputs to expose in sidebar
- createComponent() combines React component with manifest
- editorComponents array registered with DocumentRenderer
- Must be "use client" - SDK runs in browser, uses postMessage
- Component names stored in page documents - stable identifiers
- groups.ts registers component groups for palette organization
- filter option can create catch-all groups

Component Registration Pattern:
- Import createComponent and input factory functions
- Define interface for component inputs
- Create React component with ComponentProps<YourInputs>
- Register with createComponent(Component, manifest)
- Manifest: { name, label, inputs, group? }
- name uses namespace convention: "Namespace/ComponentName"

Input Types:
- createTextInput - single-line text
- createLongTextInput - multi-line text
- createNumberInput - numeric values
- createBooleanInput - toggle/checkbox
- createColorInput - color picker
- createDateInput - date/datetime picker
- createSelectInput - dropdown with options
- createRadioInput - radio button group
- createTagsInput - list of tags
- createObjectInput - nested object (sub-inputs)
- createLexicalInput - rich text editor
- createFileInput - file/media picker
- createSlotInput - slot for nesting components

Input Factory Pattern:
- name: prop name (must match interface)
- label: displayed in editor sidebar
- description: help text below label
- defaultValue: initial value
- options (select/radio): { value, label } array

Code Patterns:
```tsx
interface ComponentInputs {
  propName: string;
}

export function Component({ inputs: { propName } }: ComponentProps<ComponentInputs>) {
  return <div>{propName}</div>;
}
```

Component Groups Pattern:
```typescript
registerComponentGroup({
  name: "group-id",
  label: "Display Label",
  description: "Group description",
  filter?: (component: ComponentManifest) => boolean
});
```
