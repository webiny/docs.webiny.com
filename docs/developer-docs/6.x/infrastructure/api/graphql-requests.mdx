## Overview

GraphQL HTTP requests in Webiny flow through a simple serverless architecture consisting of Amazon CloudFront, API Gateway, and Lambda functions. The GraphQL Handler Lambda processes all queries and mutations, interacting with Amazon Cognito for authentication and DynamoDB/OpenSearch for data storage.

The flow consists of six steps:

1. The GraphQL HTTP request reaches Amazon CloudFront <diagram-letter>A</diagram-letter>.
2. CloudFront forwards the request to Amazon API Gateway <diagram-letter>B</diagram-letter>.
3. API Gateway invokes the GraphQL Handler Lambda function <diagram-letter>C</diagram-letter>.
4. The Lambda function processes the GraphQL operation and may interact with:
   - Amazon Cognito <diagram-letter>D</diagram-letter> for authentication
   - Amazon DynamoDB <diagram-letter>E</diagram-letter> for primary data storage
   - Amazon OpenSearch Service <diagram-letter>F</diagram-letter> for search queries
5. The Lambda returns an HTTP response to API Gateway <diagram-letter>B</diagram-letter>.
6. API Gateway forwards the response through CloudFront <diagram-letter>A</diagram-letter> back to the client.

## Key Points

**Single Lambda Handler:** All GraphQL operations are processed by a single Lambda function, simplifying deployment and reducing cold start overhead.

**CloudFront Caching:** CloudFront can cache certain GraphQL responses based on cache headers, improving performance for frequently accessed data.

**Authentication Layer:** Amazon Cognito handles user authentication before GraphQL operations are executed, ensuring secure access to your data.
