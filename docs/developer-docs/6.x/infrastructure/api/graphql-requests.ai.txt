Understanding GraphQL Request Architecture:
- All GraphQL operations (queries and mutations) go through a single Lambda handler
- CloudFront acts as the CDN/edge layer for global distribution
- API Gateway provides HTTP-to-Lambda integration
- The Lambda handler is responsible for all authentication, authorization, and business logic
- Cognito is used for identity authentication (JWT tokens)
- DynamoDB is the primary database; OpenSearch is used for full-text search capabilities
- The flow is consistent whether requests come from Admin Area or external clients
