Understanding Deployment Modes:
- Deployment modes only affect the API infrastructure (Lambda functions, databases)
- Admin Area are the same in both modes (CloudFront + S3)
- Mode is selected based on environment name (prod → production, anything else → development)
- The mode configuration is in the Pulumi/IaC code

Development Mode Characteristics:
- Uses AWS default VPC (created automatically for every AWS account)
- Lambda functions run in public subnets (can access internet directly)
- OpenSearch may run in single AZ (lower cost, lower availability)
- Simpler network setup (no NAT gateways, fewer route tables)
- Suitable for dev, staging, test environments

Production Mode Characteristics:
- Creates custom VPC with specific CIDR blocks
- Lambda functions run in private subnets (no direct internet access)
- OpenSearch runs in multiple AZs (high availability)
- NAT Gateway in public subnet for outbound internet access
- More complex network setup (route tables, security groups, NACLs)
- Suitable for production environment

Why Modes Exist:
- Cost: NAT Gateway and multi-AZ OpenSearch are expensive for dev/test
- Security: Production needs private subnets to meet compliance requirements
- Availability: Production needs multi-AZ to handle AZ failures
- Development: Simpler setup, faster iteration, lower cost

Mode Selection Logic:
- Environment name "prod" → production mode
- Any other name → development mode
- Can be overridden in configuration if needed
