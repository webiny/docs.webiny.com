3. No diagram — v5 had a lifecycle diagram image; not ported (no equivalent asset)
4. Implementation pattern follows DI convention: class implements `HookName.Interface`, `createImplementation()` export default
5. Import paths: `webiny/infra/core` → `CoreBeforeBuildHook`, `CoreAfterBuildHook`, etc.; same pattern for `webiny/infra/api` and `webiny/infra/admin`
6. File naming convention: PascalCase — `BeforeBuild.ts`, `AfterDeploy.ts`, `BeforeWatch.ts`
7. All three stacks (Core, Api, Admin) documented with their own code sections
8. BeforeWatch hook documented — triggered before `yarn webiny watch` starts

## Understanding

### V6 hook pattern

All hooks registered via `Infra.[Stack].Before/AfterBuild/Deploy/Watch` components in `webiny.config.tsx`:
- `src` prop points to a TypeScript file (PascalCase filename)
- File exports a default via `HookName.createImplementation()`
- Class implements `HookName.Interface` with an `execute(): Promise<void>` method
- Multiple hooks for the same event are executed in registration order
- `Ui` from `webiny/infra` is the standard dependency for logging

### Import paths

- Core hooks: `import { CoreBeforeBuildHook } from "webiny/infra/core"`
- Api hooks: `import { ApiBeforeBuildHook } from "webiny/infra/api"`
- Admin hooks: `import { AdminBeforeBuildHook } from "webiny/infra/admin"`

### Available hooks per stack

Core, Api, Admin each have: BeforeBuild, AfterBuild, BeforeDeploy, AfterDeploy, BeforeWatch

- `Infra.Admin.Pulumi` / `Infra.Api.Pulumi` / `Infra.Core.Pulumi` — covered in pulumi-extensions.mdx
