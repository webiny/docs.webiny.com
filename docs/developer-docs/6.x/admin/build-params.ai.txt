Understanding:
- Admin.BuildParam extension: at build time, generates BuildParam_<hash>.ts files + a feature.ts that imports all of them + injects BuildParamsFeature and BuildParamsInternalFeature into Extensions.tsx via RegisterFeature
- At runtime: useBuildParams() calls useFeature(BuildParamsFeature) which resolves the BuildParams abstraction from the DI container; get(key) finds the value in the registered BuildParam instances
- Import path for the hook: `import { useBuildParams } from "webiny/admin/build-params"`
- get<T>(key) returns T | null — null means key was not registered
- process.env: Vite/webpack replaces process.env.FOO at bundle time with the literal string value baked in during that specific build. This means it's NOT configurable per environment without a rebuild. BuildParams values also embed at build time but are registered in webiny.config.tsx — the central config file — rather than scattered across the codebase.
