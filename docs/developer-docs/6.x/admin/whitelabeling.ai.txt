1. Learn Webiny lesson: content/lessons/developing-with-webiny/whitelabel-extension.mdx

Key Documentation Decisions:
- Combined single-file and split-file approaches in one article (single file first as the simpler default)
- Dropped the v5 "Getting Started" scaffold component (ExtensionsGettingStarted) — not used in v6 docs
- Dropped the v5 useTags / location-based logo switching — the v6 API uses squareLogo / horizontalLogo props instead
- No screenshots — v6 docs avoid image assets unless essential
- Covers logo, title, and theme colors as one cohesive "whitelabeling" topic rather than splitting into separate articles
- Title is "Whitelabeling" because that is the developer's mental model when approaching this task

v6 Admin Branding API:
- Import: `import { AdminConfig } from "webiny/admin/configs"`
- Wrapper component: `<AdminConfig.Public>` — makes configuration available to Admin app
- `<Title value="..." />` — sets browser tab/header title
- `<Logo squareLogo={...} horizontalLogo={...} />` — two variants:
  - squareLogo: used in the collapsed/expanded sidebar
  - horizontalLogo: used on the login page
- `<Theme.Color palette="primary" color="purple" />` — overrides theme palette; valid CSS color values

File structure convention (from LW lesson):
```
extensions/
  AdminBranding/
    AdminBranding.tsx   (combined) OR
    AdminTheme.tsx      (theme only)
    AdminTitleLogo.tsx  (logo + title)
    logo.png
```

Registration in webiny.config.tsx:
```tsx
import { Admin } from "webiny/extensions";
<Admin.Extension src={"/extensions/AdminBranding/AdminBranding.tsx"} />
```

v5 vs v6 differences:
- v5: `AddLogo` component from `@webiny/app-serverless-cms`, `useTags` hook for location-based logos
- v6: `AdminConfig.Logo` with `squareLogo` / `horizontalLogo` props from `webiny/admin/configs`
- v6: registration via `<Admin.Extension>` not `export const Extension`
