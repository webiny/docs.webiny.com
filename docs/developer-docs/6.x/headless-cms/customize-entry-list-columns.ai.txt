## Key Documentation Decisions

1. Import path changed: `"webiny/admin/cms/entry/list"` (v6) not `"@webiny/app-headless-cms"` (v5)
2. Registration changed: `<Admin.Extension src={...} />` in `webiny.config.tsx`, not App.tsx modification
3. `path` prop is new/documented in v6 — allows simple field display without a custom cell component
4. `row.data.values.fieldName` is how cell renderers access field values (v6 data shape)
5. No version callouts (no "available since v5.x.x")
6. No locale content (v6 has no locales)
7. Kept all v5 column props: `name`, `header`, `modelIds`, `cell`, `sortable`, `size`, `resizable`, `visible`, `hideable`, `className`, `before`, `after`, `remove`
8. Assets copied from v5 folder — same screenshots/gifs apply visually

## V6 Code Patterns

### Import
```tsx
import { ContentEntryListConfig } from "webiny/admin/cms/entry/list";
```

### Registration in webiny.config.tsx
```tsx
<Admin.Extension src={"/extensions/contactSubmission/EmailEntryListColumn.tsx"} />
```

### Cell renderer row data access
```tsx
const { row } = useTableRow<MyRowType>();
const value = row.data.values.myField; // field values live under row.data.values
```

### Simple column with path (no custom cell needed)
```tsx
<Browser.Table.Column
    name={"email"}
    path={"values.email"}
    header={"Email"}
    modelIds={["contactSubmission"]}
/>
```
