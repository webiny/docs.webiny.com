Key Facts:
- Reference fields: can only filter by id or entryId, not by nested field values
- Objects: embed data, support filtering, data belongs only to the entry
- References: independent entries, shared across models, no nested filtering
- Never change: modelId, singularApiName, pluralApiName, fieldId, type, multipleValues
- Never remove from settings: models (ref fields), fields (object fields)
- Safe migration: add new field → migrate data → deprecate old field
