Read API:
- Read-only access to published content
- Designed for public-facing applications
- Queries only, no mutations
- Returns only published entries
- No meta field
- User fields in values object, system fields at top level
- Reference fields: can query nested values, returns null for unpublished references
- Optimized for performance and caching
- Endpoint: /cms/read

Preview API:
- Read-only access to all content (draft, published, unpublished)
- Designed for preview features and draft mode
- Queries only, no mutations
- Returns latest revision of all entries (published or draft)
- No meta field
- User fields in values object, system fields at top level
- Reference fields: can query nested values, returns complete data even for unpublished references
- Should not be cached aggressively
- Endpoint: /cms/preview

Manage API:
- Full CRUD access to all content and all revisions
- Designed for admin panels and backend integrations; used by Webiny Admin internally
- Queries and mutations
- Access to draft, published, unpublished entries; all revisions
- Includes meta field with status (published/draft/unpublished) and version number
- User fields in values object, system fields at top level
- Reference fields: returns metadata (id, entryId, modelId) only; cannot query nested values
- Never expose to frontend
- Endpoint: /cms/manage

Response Structure (all APIs):
- User-defined fields: inside values object
- System fields (id, entryId, createdOn, lastPublishedOn, etc.): top level
- meta field: Manage API only (status, version)

Authentication:
- API keys created in Settings → Access Management → API Keys
- Passed via Authorization: Bearer {key} header
- Read API key: read-only permission, safe for frontend
- Preview API key: preview or full access permission, server-side/authenticated only
- Manage API key: full access permission, server-side only, never expose publicly

Tenant Scope:
- API keys are tenant-specific in multi-tenant deployments
- Tenant ID passed via x-tenant header
- No locale scope in v6 (locales removed)

Finding API URLs:
- yarn webiny info — displays all API endpoints including CMS URLs
- API Playground in Webiny Admin — URL shown at top of playground interface

GraphQL Playground:
- Interactive GraphiQL interface
- Schema documentation explorer
- Query editor with autocomplete
- Add API key in HTTP Headers panel
- All three APIs (Read, Preview, Manage) accessible
