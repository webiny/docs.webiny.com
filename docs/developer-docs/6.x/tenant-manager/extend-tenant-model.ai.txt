TenantModelExtension Pattern:
- Import: webiny/api/tenant-manager
- Interface: TenantModelExtension.Interface
- Method: execute(extension: ModelExtension.Extension)
- Two methods available: fields() and layout()
- Factory: createImplementation({ implementation, dependencies })
- Export: default export

Extensions Field Constraint:
- All custom fields stored at tenant.extensions.*
- Cannot add fields to root tenant object
- Extensions is a container for all TenantModelExtension fields
- Structure: tenant.extensions.yourFieldName

Field Builder API:
- Same API as CMS FieldBuilder
- Available field types: text(), number(), boolean(), etc.
- Methods: label(), description(), renderer(), defaultValue()
- List support: .list() for array fields
- Predefined values: .predefinedValues([{ value, label }])

Example Fields Demonstrated:
1. websiteTitle - Simple text field
2. primaryColor - Text with default value
3. additionalColors - Text list (array)
4. font - Radio buttons with predefined options

Renderers Used:
- text-input - Single text field
- text-inputs - Multiple text fields (for lists)
- radio-buttons - Radio button group

Layout Configuration:
- Array of arrays: [["field1"], ["field2"], ["field3", "field4"]]
- Each inner array is a row
- Multiple fields in same array renders them side-by-side
- Single field per array renders full-width

Registration:
- In webiny.config.tsx
- Using <Api.Extension src={"..."} />
- Must be registered to take effect
- Path can use @/ alias for /extensions

How It Works:
- Fields automatically render in Tenant form (Admin UI)
- Fields automatically added to GraphQL schema
- Stored at tenant.extensions.fieldName in database
- Multiple extensions can be registered (all merged into extensions)
