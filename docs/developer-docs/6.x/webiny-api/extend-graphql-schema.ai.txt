Understanding:
GraphQLSchemaFactory is the extension point for adding custom queries/mutations. It uses a builder pattern:
- builder.addTypeDefs(sdl) — extends the schema with new types
- builder.addResolver({ path, dependencies, resolver }) — registers resolver with DI
- path uses dot notation: "Query.myField" or "Mutation.myMutation"
- dependencies array is positional — matches resolver factory parameters in order
- resolver factory returns the actual async resolver function (double-function pattern)
- All resolver operations return structured { data, error } responses using the Result pattern

Code Patterns:
- extend type Query / extend type Mutation to add to existing root types
- Custom response types follow { data, error } or { data, meta, error } shapes
- CmsError and CmsListMeta are built-in types available in the schema
- createImplementation wraps the class for DI registration
