Understanding:
- Api.BuildParam extension: at build time, generates a BuildParam_<hash>.ts class file in apps/api/graphql/src/buildParams/ and injects it into extensions.ts via createRegisterExtensionPlugin
- At runtime: BuildParamsImpl resolves all registered BuildParam instances (injected as a multiple dependency), then get(key) finds the matching one
- Import path: `import { BuildParams } from "webiny/api"` — also available at "webiny/api/build-params" (deprecated sub-path)
- get<T>(key) returns T | null — null means key was not registered
- Value types: string | number | boolean | object | array (Zod-validated)

- packages/project/src/extensions/ApiBuildParam.ts — extension that generates the bundle files
