Tasks SDK Understanding:
- sdk.tasks is exposed on the Webiny class as a TasksSdk instance
- All five methods delegate to standalone functions in methods/tasks/
- The triggerTask `definition` param maps to WebinyBackgroundTaskDefinitionEnum in GraphQL
- abortTask stops a running task at the next safe checkpoint (checked inside controller.runtime.isAborted())
- listLogs.where.task filters by task run ID, not definition ID
- TaskLog groups items per iteration; each item has a type field (log level/category) defined by the task itself
