Understanding:
- The Form Model is Webiny's declarative form system for the admin UI
- It uses a fluent builder API pattern similar to the CMS field builder
- Fields are defined via a registry callback, laid out with a layout builder, and rendered with FormView
- The system supports computed fields, conditional visibility/disable rules, and dynamic zones via object field templates
- All field types return chainable builders with a common set of methods plus type-specific methods
