Architecture Understanding:
- Everything is an extension/module (even core features)
- Serverless-first design (Lambda, DynamoDB, S3)
- Three main layers: Applications, Framework, Infrastructure
- Deployed to user's AWS account (not SaaS)
- Multi-tenant capable with tenant isolation

Core Components:
1. Applications: Pre-built apps (CMS, Website Builder, etc.)
2. Developer Toolkit: Framework for extending/customizing
3. Infrastructure: IaC with Pulumi for AWS resources

Key Architecture Principles:
- Modularity: Everything replaceable/extendable
- Serverless: No servers, auto-scaling, pay-per-use
- Developer Experience: Modern stack (TS, React, GraphQL)

When to Use:
Good: Enterprise, complex requirements, data control, scale
Bad: Simple sites, non-AWS, non-JS teams, quick prototypes
