Extension System Understanding:
- Extensions are modular customization pieces
- Everything in Webiny is an extension (even core features)
- Registered declaratively in webiny.config.tsx
- Implementation files live in /extensions folder
- Type-safe with full TypeScript support

Core Philosophy:
- Opt-in customization
- Sensible defaults provided
- Only extend what you need
- Platform handles the rest

Four Extension Types:
1. API Extensions - GraphQL, resolvers, lifecycle hooks, business logic
2. Admin Extensions - UI customization, branding, themes, custom pages
3. Infrastructure Extensions - Pulumi-based AWS resource modification
4. CLI Extensions - Custom CLI commands and tooling

webiny.config.tsx:
- Central configuration file
- Uses React components for configuration
- Type-safe with IDE auto-completion
- Declarative and explicit
- Supports environment-specific config

API Extensions:
- Use dependency injection (DI)
- Constructor-based dependency provision
- Type-safe DI container
- GraphQLSchemaFactory pattern
- Compile-time validation

Admin Extensions:
- Standard React components
- Can use hooks, context, React patterns
- AdminConfig for UI customization
- Regular React development workflow

Infrastructure Extensions:
- Pulumi-based AWS modifications
- Add/modify cloud resources
- Environment-specific infrastructure
- Infra.Core.Pulumi / Infra.Api.Pulumi / Infra.Admin.Pulumi

CLI Extensions:
- Custom Webiny CLI commands
- Deployment scripts, migrations, generators
- Project-specific tooling

Environment-Specific:
- Infra.Env.Is component for conditional loading
- Different configs per environment
- Production vs development resources

Pre-Built Extensions:
- Official repository: github.com/webiny/extensions
- Install via: yarn webiny extension <name>
- Adds to /extensions folder (customizable)
- Auto-registers in webiny.config.tsx
