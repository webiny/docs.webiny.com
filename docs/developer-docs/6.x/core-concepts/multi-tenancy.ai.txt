Multi-Tenancy Understanding:
- True data isolation at database level (not folders/prefixes)
- Default two-level hierarchy (root + children)
- Extensible to support complex hierarchies
- Each tenant operates like separate instance
- Business/Enterprise feature only

Core Concepts:
- Root Tenant: Administrative oversight
- Child Tenants: Isolated operational units
- Data Isolation: Complete separation of all data
- User Access: Root can access children, children isolated

Key Features:
1. Automated provisioning via API
2. Custom tenant hierarchies
3. Fine-grained access control
4. Content sharing between tenants
5. Per-tenant configurations

Common Use Cases:
- Digital agencies (client management)
- Enterprises (brand management)
- SaaS platforms (customer isolation)
- Education (department management)

Implementation Patterns:
- Enable via MultiTenancy extension
- TenantManager API for CRUD operations
- Event handlers for automation
- Custom hierarchy implementations
