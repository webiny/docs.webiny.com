## Overview

Webiny offers multiple editions to fit different project needs. All projects start with the Community edition, which is free and open source. To access advanced features like role-based access control, multi-tenancy, and publishing workflows, you can upgrade to a Business tier. For enterprise requirements like team management, SSO, and priority support, Enterprise plans are available.

## Pricing Tiers

### Community (Free)

Includes:

- Open source (MIT license)
- Website Builder, Headless CMS, and File Manager
- Perfect for personal or hobby projects

Community does **not** include:

- Role-based access control
- Multi-tenancy
- Publishing workflows
- Team management
- Single sign-on (SSO)
- Audit logs
- Priority support

### Business

Three tiers available based on team size and project requirements:

- **Business Small** ($79/mo) - For small teams and projects
- **Business Medium** ($299/mo) - Most popular, ideal for growing teams
- **Business Large** ($599/mo) - For large organizations with advanced needs

All Business tiers include everything in Community, plus:

- Role-based access control
- Multi-tenancy support
- Publishing workflows
- Production + Development license keys

### Enterprise

Custom pricing for large organizations and enterprises requiring:

- Everything in Business, plus:
- Team management
- Single sign-on (SSO)
- Audit logs
- Priority support
- Dedicated support
- Custom SLAs
- On-premises deployment options
- Custom feature development

Visit [webiny.com/pricing](https://webiny.com/pricing) to compare all features across tiers.

## Upgrade Process

### 1. Choose Your Plan

Visit [webiny.com/pricing](https://webiny.com/pricing) and select the Business tier that fits your needs. Click "Get Started" or "Upgrade now" to begin the upgrade process.

### 2. Complete Purchase

Follow the checkout process to complete your subscription. After purchase, you'll receive:

- **Project ID** - Identifies your Webiny project
- **Development API Key** - For local development and staging environments
- **Production API Key** - For production deployments

### 3. Configure Your Project

Add the project credentials to your project:

**For Local Development:**

Create or update your `.env` file in the project root:

```bash .env
WCP_PROJECT_ID=your-org/your-project-name
WCP_PROJECT_ENVIRONMENT_API_KEY=your-development-api-key
```

**For Production Deployments:**

Add the production credentials to your CI/CD provider's secrets:

```bash
# GitHub Actions, GitLab CI, or any CI provider
WCP_PROJECT_ID=your-org/your-project-name
WCP_PROJECT_ENVIRONMENT_API_KEY=your-production-api-key
```

### 4. Deploy

Redeploy your Webiny instance to activate the license:

```bash
# Local/staging deployment
yarn webiny deploy

# Production deployment (in CI/CD)
yarn webiny deploy --env=prod
```

## Verify Activation

After deployment, verify your license is active:

1. Access your Admin panel
2. Navigate to Settings or check the main menu
3. Licensed features (Role-based access control, Multi-tenancy, Publishing Workflows, etc.) should now be available

## Managing Your Subscription

### Update License Keys

If you need to update your license keys:

1. Update the `WCP_PROJECT_ID` and `WCP_PROJECT_ENVIRONMENT_API_KEY` in your `.env` or CI secrets
2. Redeploy your applications

### Different Environments

Use the appropriate API key for each environment:

- **Development/Staging**: Use the development API key
- **Production**: Use the production API key

Both API keys and the project ID are included with your Business subscription.

### Cancel or Change Plan

Manage your subscription at [webiny.com/manage](https://webiny.com/manage):

- Change between Business tiers
- Update payment information
- Cancel subscription (reverts to Community edition)

You can cancel anytime from your subscription management page.

## Feature Comparison

Visit [webiny.com/pricing](https://webiny.com/pricing) for a complete feature comparison between Community, Business, and Enterprise tiers.

Key differences:

**Community** - Core CMS features (Website Builder, Headless CMS, File Manager) with no access control, workflows, or advanced features

**Business** - Everything in Community plus role-based access control, multi-tenancy, and publishing workflows

**Enterprise** - Everything in Business plus team management, SSO, audit logs, priority support, custom features, dedicated support, and custom SLAs
