Webiny MCP Server:
- Provides Webiny-specific knowledge to AI agents
- Uses Model Context Protocol (MCP) standard
- Two tools exposed: list_webiny_skills, get_webiny_skill
- Skills contain detailed guides on Webiny APIs and patterns
- Up-to-date documentation loaded on-demand

Tools Provided:
1. list_webiny_skills - Returns catalog of available skills (name, description, context)
2. get_webiny_skill - Loads full documentation for a specific skill by name

Quick Setup Command:
- npx webiny configure-mcp <agent>
- Supported agents: claude, cursor, windsurf, copilot, cline
- Creates MCP config file (server registration)
- Creates instruction file (tells agent to use Webiny tools)
- Automatic configuration, just restart agent

Manual Setup Steps:
1. Register MCP server in agent config file
2. Add agent instructions to use Webiny tools
3. Verify server starts with MCP Inspector

Agent Config File Locations:
- Claude Code: .mcp.json (project root)
- Cursor: .cursor/mcp.json
- Windsurf: .windsurf/mcp.json
- GitHub Copilot: .vscode/mcp.json (uses "servers" key)
- Cline: .vscode/cline_mcp_settings.json
- Zed: ~/.config/zed/settings.json
- Claude Desktop: ~/Library/Application Support/Claude/claude_desktop_config.json

Agent Instruction File Locations:
- Claude Code: CLAUDE.md
- Cursor: .cursor/rules/*.mdc
- Windsurf: .windsurf/rules/*.md
- GitHub Copilot: .github/copilot-instructions.md
- Cline: System prompt in settings
- Zed: System prompt in settings

Custom Skills System:
- Built-in skills cover standard Webiny patterns
- --additional-skills flag adds custom folders (merged with built-in)
- --skills flag replaces built-in skills entirely
- Multiple folders supported, later ones override earlier
- Skills are folders containing SKILL.md file

Skill File Format (SKILL.md):
- YAML front-matter required:
  - name (required): unique identifier
  - description (required): shown in catalog
  - context (optional): groups skills, defaults to "webiny-extensions"
- Markdown content after front-matter
- Returned to agent when get_webiny_skill called

Priority Order (when skills have same name):
1. Later --additional-skills folders override earlier
2. All --additional-skills override built-in
3. --skills replaces built-in entirely

MCP Inspector:
- npx @modelcontextprotocol/inspector npx webiny mcp-server
- Connect and click List Tools
- Should see list_webiny_skills and get_webiny_skill
