Key Documentation Decisions:
1. Removed all references to Webiny Control Panel (WCP) - deprecated
2. Simplified to self-service model via webiny.com/pricing
3. Focused on API key-based activation (dev and prod keys)
4. Removed complex concepts like Organizations, Projects, etc.
5. Made it clear that both dev and prod keys come with Business subscription

Pricing Understanding:
- Community: Free, MIT license, core CMS features only
- Business Small: $79/mo - adds RBAC, multi-tenancy, workflows
- Business Medium: $299/mo (most popular) - adds RBAC, multi-tenancy, workflows
- Business Large: $599/mo - adds RBAC, multi-tenancy, workflows
- Enterprise: Custom pricing - adds team mgmt, SSO, audit logs, priority support

License Activation Process:
1. Purchase on webiny.com/pricing
2. Receive project ID and two API keys (development and production)
3. Add WCP_PROJECT_ID and WCP_PROJECT_ENVIRONMENT_API_KEY to .env (local) or CI secrets (production)
4. Deploy to activate
5. Both environments covered by single subscription

Key Differences from Old System:
- No WCP login/project creation
- No yarn webiny link-project command
- Two environment variables: WCP_PROJECT_ID and WCP_PROJECT_ENVIRONMENT_API_KEY
- Project ID + two API keys per subscription (dev + prod)
- Self-service via website

Environment Management:
- Development API key: Local dev and staging
- Production API key: Production deployments only
- Same WCP_PROJECT_ID for both environments
- Both included with Business subscription
- Set via environment variables

Required Environment Variables:
- WCP_PROJECT_ID: Format is "org-name/project-name" (e.g., "webiny-test/webiny-test-project-without-fm-td")
- WCP_PROJECT_ENVIRONMENT_API_KEY: UUID format (e.g., "d132b11d-1eb4-4766-9659-1f7b3e400d0f")
