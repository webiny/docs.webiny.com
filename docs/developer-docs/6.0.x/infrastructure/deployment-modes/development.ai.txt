Understanding Default VPC:
- Every AWS account has a default VPC in every region
- Default VPC is pre-configured with:
  - CIDR block: 172.31.0.0/16
  - One public subnet per availability zone
  - Internet Gateway attached
  - Main route table with route to Internet Gateway
- Subnets are "public" because their route table routes 0.0.0.0/0 to the Internet Gateway
- Resources in public subnets can have public IP addresses and access the internet directly

Development Mode Network Flow:
- Lambda functions → Internet Gateway → Public internet
- OpenSearch (if public) → Internet Gateway → Public internet
- External services can reach resources via public IPs (if security groups allow)

Lambda Multi-AZ:
- AWS runs Lambda functions in multiple AZs by default (not configurable)
- This provides high availability even in development mode
- If one AZ fails, Lambda requests route to other AZs automatically

OpenSearch Single-AZ:
- In development mode, OpenSearch is deployed in a single AZ (C)
- This saves cost (no replica nodes in other AZs)
- If the AZ fails, OpenSearch goes down (acceptable for dev/test)
- Production mode deploys OpenSearch across multiple AZs

Public Subnets Security:
- Resources can be accessed from the internet if security groups allow
- No NAT Gateway needed (resources have direct internet access)
- Less secure than private subnets (resources are internet-facing)
- Suitable for development where access from developer machines is needed

Availability Zones:
- Most regions have 3 AZs (us-east-1 has 6, some regions have 2)
- Default VPC creates one subnet per AZ
- Diagram shows 3 subnets (C, D, E) for 3 AZs
- Some regions only have 2 AZs, so only 2 subnets would be created
