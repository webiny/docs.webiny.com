Understanding Deployment Modes:
- Two modes: development (default VPC) and production (custom VPC)
- Mode selection is automatic based on environment name (prod = production mode)
- Development: AWS default VPC, public subnets, single AZ OpenSearch, lower cost
- Production: Custom VPC, private subnets, 3 AZs, 3 OpenSearch nodes, higher security
- Only affects API infrastructure (Core and Admin are the same across modes)
- NAT Gateway enables outbound internet from private subnets in production
- Cost difference mainly from NAT Gateway and 3x OpenSearch nodes

Development Mode Details:
- Uses AWS default VPC (automatically created for every account)
- Three public subnets across three AZs
- Lambda functions and OpenSearch can access internet directly via Internet Gateway
- OpenSearch deployed in single AZ to save costs
- Resources have direct internet access (less secure)

Production Mode Details:
- Creates custom VPC with public + private subnets
- 1 public subnet + 3 private subnets across 3 AZs
- NAT Gateway in public subnet enables outbound internet from private subnets
- Lambda and OpenSearch in private subnets (not directly accessible from internet)
- OpenSearch: 3 data nodes (one per AZ) following AWS best practices
- Higher costs: NAT Gateway data transfer + 3x OpenSearch compute/storage
