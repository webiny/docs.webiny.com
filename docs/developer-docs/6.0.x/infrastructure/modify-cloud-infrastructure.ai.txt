Key Documentation Decisions:
- v5 used per-app webiny.application.ts files with a pulumi() callback; v6 uses a single webiny.config.tsx with declarative Infra.* components + Pulumi extension files
- Dropped v5-specific sections that have no v6 equivalent: pulumiResourceNamePrefix as a callback function, env.isProduction pattern (replaced by ProductionEnvironments component), shared OpenSearch domain across accounts, CloudFront TLS example, cron job example (too long / not v6-verified)
- Kept the most universally useful examples: tagging, resource name prefix, production environments, OpenSearch, environment-conditional config, Lambda memory, IAM policy attachment
- Pulumi extension file type annotations use `any` for now — the actual Pulumi context types aren't confirmed from source
- No diff-ts code blocks — v6 docs don't use that format
- Redeploying section explains which stack to target, replacing the v5 per-app guidance

v6 Infrastructure Architecture:
- Three stacks: Core (DynamoDB, S3, OpenSearch, Cognito), API (Lambda/GraphQL), Admin (CloudFront/S3 static)
- All config in webiny.config.tsx (single file, not per-app files)
- Declarative: Infra.Aws.Tags, Infra.PulumiResourceNamePrefix, Infra.ProductionEnvironments, Infra.OpenSearch, Infra.Vpc, Infra.Env.Is
- Pulumi extensions: Infra.Core.Pulumi, Infra.Api.Pulumi, Infra.Admin.Pulumi — each takes src pointing to a .ts file

Key v5 → v6 Differences:
- v5: apps/core/webiny.application.ts + pulumi() callback inline
- v6: webiny.config.tsx + <Infra.Core.Pulumi src="/extensions/MyHandler.ts" />
- v5: pulumiResourceNamePrefix as string or callback on each app
- v6: <Infra.PulumiResourceNamePrefix prefix="myproj-" /> once globally
- v5: productionEnvironments array on each app
- v6: <Infra.ProductionEnvironments environments={["prod", "staging"]} /> once globally
- v5: tagResources() function from @webiny/pulumi-aws
- v6: <Infra.Aws.Tags tags={{...}} /> declarative component
