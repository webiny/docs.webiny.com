## Overview

By default, `prod` and `production` environments are treated as production environments and deployed using the [production deployment mode](/docs/{version}/infrastructure/deployment-modes). In this mode, Webiny deploys a more resilient and performant infrastructure configuration.

If you use additional environments that should also be treated as production (e.g. `staging`), use `Infra.ProductionEnvironments`.

## Configuration

```tsx webiny.config.tsx
import React from "react";
import { Infra } from "webiny/extensions";

export const Extensions = () => {
  return (
    <>
      <Infra.ProductionEnvironments environments={["prod", "staging"]} />
    </>
  );
};
```

## Redeploying After Changes

After making changes, redeploy by running the [`webiny deploy`](/docs/{version}/cli/deploy) command:

```
yarn webiny deploy
```

Use `--env <name>` to target a specific environment if needed.
