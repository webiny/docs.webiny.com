## Overview

By default, only the `prod` environment is treated as a production environment and deployed using the [production deployment mode](/docs/{version}/infrastructure/deployment-modes). In this mode, Webiny deploys a more resilient and performant infrastructure configuration.

If you use additional environments that should also be treated as production (e.g. `staging`), use `Infra.ProductionEnvironments`.

## Configuration

```tsx webiny.config.tsx
import React from "react";
import { Infra } from "webiny/extensions";

export const Extensions = () => {
  return (
    <>
      <Infra.ProductionEnvironments environments={["prod", "staging"]} />
    </>
  );
};
```

After making changes, redeploy all stacks for the affected environments to apply the new deployment mode:

```
yarn webiny deploy --env staging
```
