Understanding of Architecture:
- Webiny v6 deploys as serverless application to AWS
- Two main applications: API (Lambda), Admin (static React)
- API is most infrastructure-heavy (Lambda functions, DynamoDB, optional OpenSearch)
- Two database options: DynamoDB-only (simple) or DynamoDB+OpenSearch (enterprise)
- Two deployment modes affect API only:
  * Development: simpler, faster, default VPC with public subnets, single AZ OpenSearch
  * Production: Custom VPC with 3 private subnets across 3 AZs, OpenSearch with 3 data nodes, enhanced security
- Admin has same infrastructure across all environments
- Custom VPC support exists but is enterprise/advanced feature

Webiny v6 Changes from v5:
- No more "Core" application (infrastructure merged into other apps)
- No more monorepo with apps/ folder
- Single package.json at project root
- Extensions live in /extensions folder
- Simplified project structure
