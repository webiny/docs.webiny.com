## Overview

The Admin Area React application is served as static files through Amazon CloudFront and S3. When users navigate the Admin Area, their browser requests HTML, JavaScript, CSS, and other assets, which CloudFront delivers from its edge cache or retrieves from S3.

API requests from the Admin Area are routed to a separate CloudFront distribution that serves the GraphQL API.

The flow consists of three steps:

1. **Request Files:** The browser issues HTTP requests for Admin Area files (HTML, JS, CSS) to CloudFront <diagram-letter>A</diagram-letter>.
2. **Fetch from S3:** CloudFront forwards uncached requests to the S3 bucket <diagram-letter>B</diagram-letter>.
3. **Cache and Return:** CloudFront receives the file, caches it according to defined rules, and returns it to the browser.

## Caching Strategy

CloudFront applies different cache durations based on file paths:

- **`/static/*` files:** Cached for 30 days (contains versioned JavaScript, CSS, and images)
- **Other files:** Cached for 5 minutes (HTML files and other non-versioned assets)

The `/static/` folder contains content-hashed filenames (e.g., `main.a1b2c3d4.js`), making them safe to cache indefinitely. When the application is rebuilt, new hashes are generated, automatically invalidating old cached files.

## Static Folder

The `/static/` folder is created automatically during the production build process. Build tools generate unique filenames for JavaScript and CSS bundles, ensuring browsers always load the latest version after deployment.
