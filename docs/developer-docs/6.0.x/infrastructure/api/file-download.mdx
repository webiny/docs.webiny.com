## Overview

File downloads in Webiny use a dedicated Lambda function that retrieves files from S3, encodes them as base64, and returns them through API Gateway. CloudFront caches the downloaded files to improve performance and reduce Lambda invocations for frequently accessed files.

The flow consists of four steps:

1. **Request File:** The client issues a GET `/download/{file-key}` HTTP request that reaches API Gateway <diagram-letter>B</diagram-letter>.
2. **Invoke Download Lambda:** API Gateway invokes the Download Lambda function <diagram-letter>G3</diagram-letter>, passing the file key.
3. **Fetch and Encode:** The Lambda function retrieves the file from S3 <diagram-letter>G1</diagram-letter>, encodes it as base64, and returns it to API Gateway <diagram-letter>B</diagram-letter>.
4. **Transform and Cache:** API Gateway transforms the base64 string to binary and sends it to CloudFront <diagram-letter>A</diagram-letter>, which caches and returns it to the client.

## Binary Handling

API Gateway uses [binary media type handling](https://docs.aws.amazon.com/apigateway/latest/developerguide/api-gateway-payload-encodings.html) to transform base64-encoded responses from Lambda into actual binary files. This allows Lambda functions (which work with text payloads) to serve binary content.

## Caching Strategy

CloudFront caches downloaded files, reducing the number of Lambda invocations and S3 requests. Cached files are served directly from CloudFront edge locations, improving download speed for users worldwide.
