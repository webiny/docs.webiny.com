## Overview

Webiny uses Amazon S3 pre-signed POST payloads for file uploads, allowing clients to upload files directly to S3 without routing binary data through Lambda functions. This approach is more efficient and cost-effective than traditional multipart uploads through API Gateway and Lambda.

The File Manager application generates pre-signed POST data through a GraphQL mutation, the client uploads directly to S3, and then a final GraphQL mutation stores the file metadata.

The flow consists of three steps:

1. **Generate Pre-signed POST Data:** The client issues a GraphQL mutation to the GraphQL Handler <diagram-letter>C</diagram-letter>, which generates pre-signed POST data for S3.
2. **Upload to S3:** The client uses the pre-signed POST data to upload the file directly to the S3 bucket <diagram-letter>G1</diagram-letter> via a POST HTTP request.
3. **Store Metadata:** The client issues another GraphQL mutation to store file metadata in Amazon DynamoDB <diagram-letter>E</diagram-letter> and Amazon OpenSearch Service <diagram-letter>F</diagram-letter>.

## Security

**Authentication and Authorization:** Both GraphQL requests (steps 1 and 3) require proper authentication via Amazon Cognito. Only users with appropriate permissions can upload files.

**Pre-signed URL Expiration:** Pre-signed POST URLs expire after a short time window, preventing unauthorized uploads.

**Bucket Policies:** The S3 bucket uses strict policies to accept uploads only with valid pre-signed POST data.

## Configuration

Maximum file upload size is configurable through the File Manager's settings in the Admin Area. This setting is enforced both client-side and server-side.
