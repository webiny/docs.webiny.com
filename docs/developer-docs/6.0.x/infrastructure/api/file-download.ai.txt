Understanding File Download Architecture:
- File downloads use a different path than GraphQL requests: GET /download/{file-key}
- The Download Lambda (G3) is separate from the GraphQL Handler (C)
- Lambda functions in API Gateway can only return text payloads, so binary files must be base64-encoded
- API Gateway has built-in support for transforming base64 → binary based on content-type headers
- CloudFront acts as a CDN, caching files at edge locations worldwide
- The file-key is extracted from the URL path and used to fetch from S3
- No authentication is typically required for downloads - access control happens during upload or through pre-signed URLs

Binary Encoding Flow:
1. Lambda retrieves binary file from S3
2. Lambda encodes file as base64 string
3. Lambda returns base64 string with appropriate content-type header
4. API Gateway sees content-type header and automatically decodes base64 → binary
5. CloudFront receives binary, caches it, sends to client

Caching Benefits:
- Reduces Lambda invocations → lower cost
- Reduces S3 requests → lower cost
- Faster downloads from edge locations → better UX
- Reduces load on origin infrastructure
