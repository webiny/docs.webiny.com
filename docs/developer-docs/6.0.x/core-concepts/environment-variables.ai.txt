Environment Variable System:
- Prefix-based routing (WEBINY_ADMIN_, WEBINY_API_)
- Application isolation for security
- Optional for most development
- Loaded from .env file locally
- Set via export or CI/CD for production

Key Prefixes:
- WEBINY_ADMIN_ - Only in Admin app
- WEBINY_API_ - Only in API app
- No prefix - Available everywhere (discouraged)

Common Use Cases:
1. Third-party service credentials
2. Feature flags
3. External API configuration
4. Analytics and monitoring
5. Email service setup

Best Practices:
- Never commit .env files
- Document in .env.example
- Use consistent naming
- Minimize shared variables
- Type definitions in webiny-env.d.ts

Migration Notes:
- v5 used REACT_APP_ prefix
- v6 uses WEBINY_ADMIN_ prefix
- Update code when migrating
