Project Structure Understanding:
- Single package.json (not a monorepo)
- All custom code in extensions/ folder
- webiny.config.tsx is the entry point
- Clean separation of framework vs custom code
- Standard JS/TS tooling (ESLint, TSConfig)

Key Directories:
- extensions/ - All custom code
- public/ - Admin app static assets
- node_modules/ - Dependencies (gitignored)

Key Files:
- webiny.config.tsx - Central configuration
- package.json - Dependencies
- tsconfig.json - TypeScript config
- webiny-env.d.ts - Type definitions

Extension Organization:
- Can organize by type (api/, admin/, etc.)
- Can organize by feature (ecommerce/, blog/, etc.)
- Referenced by path in webiny.config.tsx
- All paths relative to project root

Best Practices:
- Group related functionality
- Use consistent naming conventions
- Don't store build outputs in git
- Keep extensions portable
