Local Development Understanding:
- Admin: Standard React dev server with HMR
- API: Local AWS Lambda Development (unique approach)
- Infrastructure: Requires deployment, no local testing
- Hybrid model: Local code, cloud services

Watch Commands:
- `yarn webiny watch admin` - Port 3001, React dev
- `yarn webiny watch api` - Lambda stub forwarding
- Can run concurrently in separate terminals
- Must redeploy API after stopping watch

Local AWS Lambda Development:
1. Deploy stubs to Lambda functions
2. Forward events to local machine
3. Execute code locally with Lambda context
4. Return response through stub
5. See changes without deployment

Environment Variables:
- Prefix with WEBINY_ADMIN_ or WEBINY_API_
- Application-specific isolation
- Optional for most development
- Document in .env.example

Key Workflows:
- Initial: Clone → yarn → deploy core api → watch
- Daily: watch → develop → test → deploy when ready
- Team: Simple onboarding, no complex setup
