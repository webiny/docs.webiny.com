Debugging Understanding:
- Three error types: API, Admin, Infrastructure
- Different tools for different applications
- Local development vs production debugging
- Importance of identifying error origin first

API Debugging:
- console.log for local development
- Logger for production (CloudWatch)
- Logs appear in terminal during watch
- Structured logging recommended
- Log levels: debug, info, warn, error

Admin Debugging:
- Browser DevTools (Console, Network, Elements)
- React Developer Tools extension
- GraphQL Network Inspector extension
- Standard React debugging workflow
- Element selection for component discovery

Infrastructure Debugging:
- Pulumi error messages in terminal
- CloudFormation stack events
- AWS service quotas and limits
- IAM permission issues
- Configuration validation

Tools Overview:
API:
- console.log (local)
- Logger (production)
- CloudWatch Logs

Admin:
- Browser Console
- Network Tab
- React DevTools
- GraphQL Inspector
- Elements panel

Infrastructure:
- Terminal output
- Pulumi error messages
- AWS Console

Debugging Workflows:
API: watch → trigger → check logs → fix → test
Admin: watch → DevTools → reproduce → inspect → fix → test
Production: CloudWatch (API) / Error tracking (Admin)

Best Practices:
- Use structured logging
- Log errors with context
- Appropriate log levels
- Remove debug logs before production
- Use React DevTools for component inspection

Common Issues:
Admin:
- Component not rendering
- GraphQL errors
- Styling issues

Infrastructure:
- Permission errors
- Resource limits
- Configuration errors
