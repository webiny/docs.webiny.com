AI Development Understanding:
- AI as teammate/accelerator, not autopilot
- Most effective with clear context and focused tasks
- Developers own architectural decisions and consequences
- Review AI output as carefully as human-written code
- Works best for implementation acceleration, not design

Where AI Helps:
- Extension scaffolding
- GraphQL generation
- TypeScript boilerplate
- Code exploration and explanation
- Debugging assistance
- Utility functions
- UI component drafts

Where Developers Lead:
- Architecture choices
- Security and access control
- Multi-tenancy behavior
- Business logic validation
- Maintainability decisions
- Testing strategies
- Production readiness

Effective Prompting:
- Provide context before asking for code
- Be specific about requirements
- Define constraints clearly
- Request platform-native solutions
- Break large tasks into steps
- Include expected output format

Code Review Checklist:
- Proper placement in project structure
- Follows existing patterns
- No unnecessary duplication
- Respects tenant boundaries
- Maintainable and upgradeable
- Security implications checked

Common Pitfalls:
- Too large implementations
- Insufficient context
- Accepting without review
- Inventing patterns vs following existing
- Wrong placement in system
- Treating prototypes as production code
