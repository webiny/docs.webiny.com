SDK Understanding:
- TypeScript library for external app integration
- Programmatic interface to Webiny instance
- Convenience layer over GraphQL API
- Type-safe with generic support
- Result pattern for error handling

When to Use SDK:
- External applications (Next.js, Vue, SvelteKit, Node.js)
- NOT for Admin extensions or API extensions
- Frontend/backend apps outside Webiny
- Any app needing to read/write Webiny content

Installation:
- npm install @webiny/sdk
- Initialize once with credentials
- Reuse singleton instance

Three Main Modules:
1. sdk.cms - Headless CMS operations
2. sdk.fileManager - File/folder management
3. sdk.websiteBuilder - Website content access

Authentication:
- API key token from Admin (Settings → API Keys)
- Configurable permissions per app
- Required for all SDK calls

Core Concepts:
1. Result Pattern - isOk() check, no throwing
2. Fields Parameter - Request specific fields only
3. TypeScript Generics - Type-safe responses
4. Depth Parameter - Control reference resolution

Result Pattern:
- Every method returns Result object
- Check success: result.isOk()
- Access data: result.value
- Access error: result.error
- Explicit error handling

Best Practices:
- Store credentials in env vars
- Create singleton SDK instance
- Define interfaces for content models
- Use generics for type safety
- Select only needed fields
- Always handle errors
