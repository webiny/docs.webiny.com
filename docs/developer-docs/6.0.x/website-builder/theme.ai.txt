Understanding Website Builder Theme System:
- Three-file system: theme.css (define), theme.ts (register), tailwind.css (bridge)
- Two-step pattern for editor-selectable styles: CSS variable → theme.ts registration
- theme.css defines CSS custom properties and typography classes
- theme.ts exports theme object via createTheme() - passed to contentSdk.init()
- tailwind.css bridges WB variables to Tailwind tokens (bg-primary, text-primary)
- __THEME_CSS__ global provided by webpack plugin injectThemeCss
- Fonts array in theme.ts injects fonts into editor iframe
- clamp() for fluid responsive typography
- Font weight mismatch common issue - layout.tsx vs fonts array
- Editor device controls for responsive preview
- Typography sizing differs between canvas and sidebar (expected with clamp/rem)

Theme System Pattern:
- Editor-selectable styles: define in theme.css, register in theme.ts
- Component-only styles: use Tailwind/CSS directly, no theme.ts entry
- Color variables use semantic names (--wb-theme-color-primary)
- Typography classes use clamp() for fluid sizing
- colors array populates editor color picker
- typography object populates editor toolbar

Code Patterns:
- CSS variables: --wb-theme-color-[semantic-name]
- Typography classes: .wb-[element]-[variant]
- clamp() syntax: clamp(min, fluid, max)
- createTheme() receives: css, fonts, colors, typography
- colors entry: { id, label, value }
- typography entry: { id, label, tag, className }

Font Loading Pattern:
1. layout.tsx - Next.js font config
2. tailwind.css - --font-sans token
3. theme.css - --wb-theme-font-family variable
4. theme.ts - fonts array for editor iframe
