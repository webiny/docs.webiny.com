Understanding Website Builder Setup:
- Editor loads Next.js app in iframe during editing
- Components and styles live entirely in Next.js project
- Webiny stores only page structure and component inputs
- @webiny/website-builder-nextjs SDK handles editor ↔ app communication
- Starter kit at github.com/webiny/website-builder-nextjs
- SDK versions must match Webiny version
- Three env vars needed: API_KEY, API_HOST, API_TENANT
- API key auto-created by Website Builder (read-only)
- Configure Next.js dialog in Admin provides all credentials
- Catch-all route [[...slug]]/page.tsx renders all pages
- Preview API route enables draft mode for unpublished pages
